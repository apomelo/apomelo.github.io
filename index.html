Java JDK Upgrade

发表于 2025-09-19 分类于 java 阅读次数：

简介

本文梳理了 Java 8 到 Java 25 各版本的核心特性演进，帮助开发者了解 Java 的发展脉络，为 JDK 升级决策提供参考。

为什么要升级 JDK？

许多项目仍在使用 Java 8，但新版本 JDK 带来了显著的改进：

维度 升级收益
性能 新版 GC（ZGC/Shenandoah）实现亚毫秒级暂停，JIT 编译优化提升 10-20% 吞吐量
并发 虚拟线程（Java 21）支持百万级轻量线程，彻底改变高并发编程模型
语法 var、Record、模式匹配、文本块等特性大幅减少样板代码
安全 持续的安全补丁和漏洞修复，旧版本已停止公开更新
生态 主流框架（Spring 6+、Quarkus、Micronaut）要求 Java 17+

推荐升级路径

  • Java 8 → Java 17：稳定 LTS，企业首选，Spring Boot 3.x 最低要求
  • Java 17 → Java 21：获得虚拟线程，高并发场景强烈推荐
  • Java 21 → Java 25：最新 LTS，作用域值 + 内存优化

LTS 版本支持周期

版本 发布时间 Oracle 扩展支持 推荐场景
Java 8 2014.03 2030.12 遗留系统维护
Java 11 2018.09 2032.01 保守升级选择
Java 17 2021.09 2029.09 当前企业主流
Java 21 2023.09 2031.09 新项目首选
Java 25 2025.09 2033.09 前沿技术探索

opentelemetry

发表于 2024-06-04 分类于 default 阅读次数：

简介

OpenTelemetry 是一个开源的可观察性框架，旨在帮助开发者收集、处理和导出软件应用中的度量指标、日志和分布式追踪信息。它是由 OpenTracing 和 OpenCensus 项目合并而来的，目的是创建一个统一的、标准化的工具集，简化可观察性数据的收集和分析。

OpenTelemetry 的主要功能

  1. 分布式追踪（Distributed Tracing）：
    • 追踪请求在分布式系统中的传播路径，帮助识别性能瓶颈和问题。
  2. 度量指标（Metrics）：
    • 收集和分析系统性能指标，如响应时间、请求数、错误率等。
  3. 日志（Logging）：
    • 记录和分析应用运行时的日志信息，帮助诊断和调试问题。

OpenTelemetry 的组成部分

  1. API：
    • 提供标准化的接口供开发者使用，用于生成和收集追踪、度量和日志数据。
  2. SDK：
    • 提供具体的实现，负责数据的采集和传输。
  3. Exporters：
    • 用于将收集的数据导出到各种后端系统，如 Jaeger、Prometheus、ElasticSearch 等。
  4. Instrumentation Libraries：
    • 提供开箱即用的库，用于自动化数据收集，无需修改应用代码。

使用场景

  • 性能监控：
    • 通过追踪请求路径和分析性能指标，找出系统中的性能瓶颈。
  • 故障排除：
    • 通过分布式追踪和日志分析，快速定位问题根源。
  • 系统优化：
    • 基于度量数据，优化系统资源使用和响应速度。

如何使用 OpenTelemetry

  1. 集成 SDK：
    • 根据使用的编程语言（如 Java、Python、Go 等）选择合适的 OpenTelemetry SDK。
  2. 配置 Instrumentation：
    • 使用自动化的 Instrumentation 库来收集基本的可观察性数据。
  3. 设置 Exporters：
    • 配置数据导出目标，将收集到的数据发送到所选的后端系统进行存储和分析。
  4. 分析和可视化：
    • 使用相应的工具（如 Grafana、Kibana）对导出的数据进行分析和可视化。

OpenTelemetry 的目标是提供一个统一、灵活且易于扩展的解决方案，以满足现代分布式系统对可观察性数据的需求。

prometheus

发表于 2024-05-28 分类于 prometheus 阅读次数：

简介

Prometheus 是一个开源系统监控和报警工具包，最早由 SoundCloud 开发，现在是 CNCF（Cloud Native Computing Foundation）的一个项目。它的设计目标是可靠性和高效性，特别适用于动态云环境下的系统和服务监控。

核心特性

  1. 多维数据模型： Prometheus 使用多维数据模型，通过时间序列来存储数据。每个时间序列由一个度量名称和一组键值对的标签来标识。
  2. 强大的查询语言（PromQL）： Prometheus 提供了一种灵活而强大的查询语言，PromQL，用于实时查询和分析存储的数据。
  3. 无需依赖分布式存储： Prometheus 的设计目标是通过单个服务器节点提供可靠的服务，这使其配置和管理更为简单。尽管它可以通过联邦集成多个 Prometheus 服务器，但它本身不依赖于分布式存储系统。
  4. 高效的数据存储： 数据以时间序列的形式存储，并通过内存和磁盘的组合方式进行高效存储和检索。
  5. 可视化支持： Prometheus 提供内置的图形和仪表盘功能，同时与 Grafana 等外部可视化工具高度集成。
  6. 灵活的告警管理： Prometheus 自带 Alertmanager，可以管理告警规则，处理告警并将其路由到不同的通知渠道（如电子邮件、Slack、PagerDuty 等）。

工作原理

Prometheus 通过 HTTP 协议定期抓取被监控目标的指标数据。这些目标可以是物理主机、虚拟机、数据库、应用程序等。被监控目标需要暴露一个 /metrics 端点，Prometheus 会通过这个端点获取指标数据。

基本组件

  1. Prometheus 服务器： 负责抓取和存储时间序列数据，处理查询请求。
  2. Exporters（导出器）： 用于暴露指标数据的工具。不同类型的系统和应用程序可能需要不同的导出器。例如，Node Exporter 用于暴露 Linux 系统的硬件和操作系统指标。
  3. Pushgateway： 用于短期任务的指标推送。通常情况下，Prometheus 是通过 pull 模型抓取数据，但对于某些短期任务，推送模型更为合适。
  4. Alertmanager： 处理来自 Prometheus 服务器的告警，支持告警的分组、抑制和路由，并将告警发送到各种通知渠道。
  5. 客户端库： 提供给应用程序开发者用于在代码中直接暴露自定义指标。Prometheus 提供多种语言的客户端库，包括 Go、Java、Python 等。

典型使用场景

  1. 系统监控： 监控主机和容器的性能指标，如 CPU 使用率、内存使用率、磁盘 I/O 等。
  2. 应用程序监控： 监控应用程序的性能和健康状况，例如 HTTP 请求的响应时间、错误率、数据库查询时间等。
  3. 业务指标监控： 监控业务相关的指标，例如网站的活跃用户数、订单处理量、支付成功率等。

VBA(Visual Basic for Applications) 笔记

简介

VBA，全名 Visual Basic for Applications，是 Microsoft 为其 Office 系列产品如 Excel、Word、Access 等提供的一种宏语言。它是 Visual Basic 的一个子集，主要用于自动化 Office 应用程序的任务，包括创建用户定义的函数、自动执行复杂任务、创建和管理 Windows 应用程序等。

VBA 是一种面向对象的编程语言，可以创建复杂的用户界面和其他高级功能。它具有强大的编程能力，可以访问 Windows API 和其他低级功能，同时还可以使用 SQL 连接数据库。

VBA 的主要优点是易于学习和使用，尤其是对于那些已经熟悉 Microsoft Office 的用户。此外，由于它是 Office 的内置组件，因此无需另外安装就可以使用。然而，它的缺点是运行速度相对较慢，且只能在 Windows 环境下运行。

简介

splunk 是什么?

Splunk 是一款用于搜索、监控和分析大规模实时生成的日志和数据的软件。它可以帮助组织从日志、事件和其他数据源中提取有价值的信息，用于监控系统状态、分析应用程序性能、发现安全事件等用途。Splunk 提供了强大的搜索和查询功能，可以帮助用户快速准确地找到需要的信息，并提供可视化和报告工具，用于展示和分析数据趋势、模式和异常。

Splunk 可以与各种数据源集成，包括服务器日志、应用程序日志、网络数据、传感器数据等，支持多种数据格式和协议。它还提供了丰富的应用程序和插件，用于扩展和定制功能，满足不同领域和行业的需求。

Splunk 是一款强大、灵活、易于使用的日志和数据分析工具，广泛应用于 IT 运维、安全监控、业务分析等领域，帮助组织实现数据驱动的决策和运营管理。

jetty 笔记

发表于 2024-03-12 分类于 java 阅读次数：

简介

Jetty 是一个基于 Java 的开源的 Web 服务器和 Servlet 容器，它是由 Eclipse Foundation 托管的一个项目。Jetty 提供了一个灵活、可嵌入的服务器组件，可以用于构建各种类型的 Java Web 应用程序，包括传统的基于 Servlet 的 Web 应用、RESTful 服务、WebSocket 应用等。

Jetty 的特点和优势包括：

  1. 轻量级和高性能： Jetty 是一个轻量级的 Web 服务器，具有卓越的性能。它采用异步 I/O 和非阻塞的处理模型，能够高效地处理大量的并发请求。
  2. 嵌入式容器： Jetty 提供了一个嵌入式的容器，可以将 Jetty 嵌入到应用程序中，作为应用程序的一部分运行，而不需要单独部署一个独立的服务器。这使得 Jetty 在开发和测试阶段非常方便，并且能够灵活地与应用程序集成。
  3. 支持标准： Jetty 是一个符合标准的 Servlet 容器，完全支持 Java EE 规范，包括 Servlet、WebSocket、JSP 等规范。因此，它可以无缝地与其他 Java EE 组件集成，如 JPA、CDI 等。
  4. 灵活的配置和扩展性： Jetty 提供了丰富的配置选项和可扩展的 API，可以根据需要灵活地配置和定制服务器。它支持多种插件和扩展，可以轻松地扩展功能，满足不同的需求。
  5. 可靠性和稳定性： Jetty 是一个成熟的开源项目，经过了长期的发展和实践验证，具有较高的可靠性和稳定性。它广泛应用于生产环境中的大型系统和互联网应用，并且得到了很好的支持和社区反馈。

总的来说，Jetty 是一个功能强大、灵活、性能优异的开源 Web 服务器和 Servlet 容器，适用于构建各种类型的 Java Web 应用程序，是 Java 开发人员的重要选择之一。

docker 介绍

发表于 2024-03-05 分类于 tool 阅读次数：

简介

Docker 是一个开源的应用容器引擎，用于轻松地打包、发布和运行应用程序。它可以将应用程序及其依赖项打包为一个独立的容器，使应用程序能够在任何环境中都能以相同的方式运行。

以下是 Docker 的一些主要特点和优势：

  1. 轻量级的容器化： Docker 使用容器化技术，每个容器都是一个独立的运行时环境，包含了应用程序及其依赖项。与虚拟机相比，容器更轻量级，启动速度更快，资源占用更少。
  2. 跨平台性： Docker 容器可以在任何支持 Docker 的平台上运行，无论是开发环境、测试环境还是生产环境。这使得开发、测试和部署过程更加一致和可靠。
  3. 快速部署和扩展： Docker 容器可以快速部署，并且可以根据需要快速扩展。通过 Docker 容器，您可以更快地将应用程序部署到生产环境中，并根据负载的变化动态调整容器数量。
  4. 一致的开发环境： 使用 Docker 可以创建具有相同软件版本和配置的开发环境，这有助于团队成员之间更加一致地开发和测试应用程序，避免了“在我的机器上可以运行”的问题。
  5. 容器编排和管理： Docker 提供了一系列工具和平台，如 Docker Compose、Docker Swarm 和 Kubernetes，用于容器编排和管理。这些工具使得在生产环境中管理和调度大规模容器集群变得更加简单和可靠。
  6. 持续集成和持续部署： Docker 可以与持续集成和持续部署工具集成，如 Jenkins、GitLab CI、Travis CI 等，实现自动化构建、测试和部署应用程序的流程。

总的来说，Docker 提供了一种轻量级、灵活、可移植和可扩展的容器化解决方案，使得开发人员和运维人员能够更高效地构建、发布和运行应用程序。
