docker 介绍

发表于 2024-03-05 分类于 tool 阅读次数：

简介

Docker 是一个开源的应用容器引擎，用于轻松地打包、发布和运行应用程序。它可以将应用程序及其依赖项打包为一个独立的容器，使应用程序能够在任何环境中都能以相同的方式运行。

以下是 Docker 的一些主要特点和优势：

  1. 轻量级的容器化： Docker 使用容器化技术，每个容器都是一个独立的运行时环境，包含了应用程序及其依赖项。与虚拟机相比，容器更轻量级，启动速度更快，资源占用更少。
  2. 跨平台性： Docker 容器可以在任何支持 Docker 的平台上运行，无论是开发环境、测试环境还是生产环境。这使得开发、测试和部署过程更加一致和可靠。
  3. 快速部署和扩展： Docker 容器可以快速部署，并且可以根据需要快速扩展。通过 Docker 容器，您可以更快地将应用程序部署到生产环境中，并根据负载的变化动态调整容器数量。
  4. 一致的开发环境： 使用 Docker 可以创建具有相同软件版本和配置的开发环境，这有助于团队成员之间更加一致地开发和测试应用程序，避免了“在我的机器上可以运行”的问题。
  5. 容器编排和管理： Docker 提供了一系列工具和平台，如 Docker Compose、Docker Swarm 和 Kubernetes，用于容器编排和管理。这些工具使得在生产环境中管理和调度大规模容器集群变得更加简单和可靠。
  6. 持续集成和持续部署： Docker 可以与持续集成和持续部署工具集成，如 Jenkins、GitLab CI、Travis CI 等，实现自动化构建、测试和部署应用程序的流程。

总的来说，Docker 提供了一种轻量级、灵活、可移植和可扩展的容器化解决方案，使得开发人员和运维人员能够更高效地构建、发布和运行应用程序。

镜像选择

Docker 可以在各种操作系统上运行，并且可以使用不同的操作系统作为基础镜像来构建自定义镜像。一般来说，以下几种操作系统常用于作为 Docker 镜像的基础：

  1. Linux: Linux 是 Docker 最常用的基础操作系统之一，特别是一些轻量级的 Linux 发行版，如 Alpine Linux、Ubuntu、Debian 等。这些 Linux 发行版通常具有较小的映像大小和较低的资源消耗，非常适合于容器化应用。
  2. Windows: Docker 也支持在 Windows 操作系统上运行，因此可以使用 Windows Server Core 或 Windows Nano Server 作为基础镜像。Windows Server Core 镜像提供了完整的 Windows Server 环境，而 Nano Server 镜像则提供了更轻量级的 Windows 环境。
  3. macOS: 尽管 macOS 通常用于开发环境，但也可以通过 Docker Desktop 在 macOS 上运行 Docker 容器。在 macOS 上使用的 Docker 镜像通常是基于 Linux 的，因为 Docker Desktop 在 macOS 上使用了一个 Linux 虚拟机来运行 Docker 引擎。
  4. 其他操作系统: Docker 还支持在其他操作系统上运行，如 FreeBSD。但是在实践中，Linux 是最常用的基础操作系统之一，因为它具有广泛的软件支持和社区资源。

总的来说，您可以根据您的具体需求选择适合的操作系统作为 Docker 镜像的基础，以满足您的应用程序的要求和环境。

dockerfile

Dockerfile 是一个文本文件，用于定义 Docker 镜像的配置和构建步骤。通过 Dockerfile，可以指定镜像的基础映像、安装软件、设置环境变量、暴露端口、拷贝文件等操作，以及构建所需的环境。以下是 Dockerfile 的一般结构和主要指令：

在 Dockerfile 中，每个指令都会创建一个新的镜像层，构建镜像时会依次执行这些指令，最终生成一个完整的镜像。下面是对 Dockerfile 中一些常用指令的解释：

  • FROM：指定基础映像，Docker 将基于这个基础映像构建新的镜像。FROM 指令必须是 Dockerfile 的第一个指令。
  • LABEL：用于设置镜像的作者信息等标签。
  • ENV：设置环境变量，这些变量可在后续指令中使用。
  • WORKDIR：设置工作目录，后续的指令将在这个目录中执行。
  • COPY 和 ADD：将文件或目录从本地系统复制到容器中。
  • RUN：运行命令，用于在容器中安装软件、配置环境等操作。
  • EXPOSE：暴露容器运行时的端口，但并不会将该端口映射到宿主机端口。
  • CMD 和 ENTRYPOINT：定义容器启动时要运行的命令。CMD 用于指定命令的默认参数，ENTRYPOINT 用于设置容器的入口点。

安装 wsl 以及 linux 子系统

需要先安装 wsl 以及 linux 子系统

输入 wsl –install 命令，然后重启计算机。此命令将启用运行 WSL 并安装 Linux 的 Ubuntu 发行版所需的功能。

仅当根本没有安装 WSL 时，上述命令才有效。 如果运行 wsl --install 并查看 WSL 帮助文本，请尝试运行 wsl --list --online 以查看可用发行版列表并运行 wsl --install -d <DistroName> 以安装一个发行版。

更改默认安装的 Linux 发行版

默认情况下，安装的 Linux 分发版为 Ubuntu。 可以使用 -d 标志进行更改。

  • 若要更改安装的发行版，请输入：wsl --install -d <Distribution Name>。 将 <Distribution Name> 替换为要安装的发行版的名称。
  • 若要查看可通过在线商店下载的可用 Linux 发行版列表，请输入：wsl --list --online 或 wsl -l -o。
  • 若要在初始安装后安装其他 Linux 发行版，还可使用命令：wsl --install -d <Distribution Name>。

提示
如果要通过 Linux/Bash 命令行（而不是通过 PowerShell 或命令提示符）安装其他发行版，必须在命令中使用 .exe：wsl.exe --install -d <Distribution Name> 或若要列出可用发行版，则使用：wsl.exe -l -o。

检查正在运行的 WSL 版本

检查每个发行版的 WSL 版本：wsl -l -v

要在安装新的 Linux 发行版时将默认版本设置为 WSL 1 或 WSL 2，请使用命令 wsl --set-default-version <Version#>，将 <Version#> 替换为 1 或 2。

要设置与 wsl 命令一起使用的默认 Linux 发行版，请输入 wsl -s <DistributionName> 或 wsl --set-default <DistributionName>，将 <DistributionName> 替换为要使用的 Linux 发行版的名称。

要在 PowerShell 或 Windows 命令提示符下运行特定的 WSL 发行版而不更改默认发行版，请使用命令 wsl -d <DistributionName>，将 <DistributionName> 替换为要使用的发行版的名称。

有关详细信息，请参阅 WSL 的基本命令指南。
