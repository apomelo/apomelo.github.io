简介

splunk 是什么?

Splunk 是一款用于搜索、监控和分析大规模实时生成的日志和数据的软件。它可以帮助组织从日志、事件和其他数据源中提取有价值的信息，用于监控系统状态、分析应用程序性能、发现安全事件等用途。Splunk 提供了强大的搜索和查询功能，可以帮助用户快速准确地找到需要的信息，并提供可视化和报告工具，用于展示和分析数据趋势、模式和异常。

Splunk 可以与各种数据源集成，包括服务器日志、应用程序日志、网络数据、传感器数据等，支持多种数据格式和协议。它还提供了丰富的应用程序和插件，用于扩展和定制功能，满足不同领域和行业的需求。

Splunk 是一款强大、灵活、易于使用的日志和数据分析工具，广泛应用于 IT 运维、安全监控、业务分析等领域，帮助组织实现数据驱动的决策和运营管理。

服务架构

Splunk 是一个用于搜索、监控和分析大规模数据的平台，其架构可以根据需求进行扩展和定制。以下是 Splunk 的典型服务架构：

  • Universal Forwarder：
    • 功能：Universal Forwarder 是 Splunk 中的轻量级代理程序，负责收集和转发数据。
    • 工作原理：Universal Forwarder 安装在数据源服务器上，监视本地文件、网络流量或其他数据源，并将数据转发到 Splunk 索引器或集群。
    • 特点：它具有低资源消耗、高性能和配置灵活性的特点，可以定制为满足不同数据源类型和需求。
  • Heavy Forwarder：
    • 功能：Heavy Forwarder 是 Universal Forwarder 的增强版，具有更多的功能和处理能力。
    • 工作原理：与 Universal Forwarder 类似，但 Heavy Forwarder 具有更多的数据处理和转换功能，例如字段提取、日志格式解析等。
    • 特点：适用于需要在数据转发前对数据进行预处理或转换的场景，如数据清洗、格式转换等。
  • Indexer：
    • 功能：Indexer 负责接收、索引和存储数据，以便后续的搜索和分析。
    • 工作原理：当数据到达 Indexer 时，它将数据进行索引并存储在本地磁盘上，以支持后续的搜索查询。
    • 特点：具有高度可扩展性，可以构建成集群以提高性能和可用性。
  • Search Head：
    • 功能：Search Head 是用于执行搜索、分析和可视化数据的用户界面。
    • 工作原理：用户通过 Web 界面与 Search Head 交互，执行搜索查询、查看报表、创建仪表盘等操作。
    • 特点：支持高级搜索查询语言、可视化图表和报表功能，提供丰富的数据分析和可视化能力。
  • Deployment Server：
    • 功能：Deployment Server 用于集中管理 Splunk 部署的配置和应用程序。
    • 工作原理：管理员可以使用 Deployment Server 将配置文件、应用程序和更新传输到 Splunk 环境中的所有组件，以确保部署的一致性和管理。
    • 特点：支持集中化管理和配置自动化，可以根据需要轻松部署和管理大规模 Splunk 环境。
  • Cluster Master：
    • 功能：Cluster Master 用于管理和协调 Indexer 集群中的数据复制、索引管理和集群配置。
    • 工作原理：Cluster Master 监视索引器的状态、执行数据复制和重新平衡任务，并管理集群中的拓扑结构和配置。
    • 特点：支持高可用性、自动化和集群优化，可以实现集群中索引器的自动发现和管理。
  • License Master：
    • 功能：License Master 用于管理 Splunk 环境中的许可证。
    • 工作原理：License Master 负责分配和管理许可证，监视许可证使用情况，并确保环境中的所有组件都具有有效的许可证。
    • 特点：支持许可证的集中管理和监控，确保 Splunk 环境的合规性和许可证使用效率。

Common Stats Functions

Common statistical functions used with the chart, stats, and timechart commands. Field names can be wildcarded, so avg(*delay) might calculate the average of the delay and xdelay fields.

Function Description
avg(X) Returns the average of the values of field X.
count(X) Returns the number of occurrences of the field X. To indicate a specific field value to match, format X as eval(field=”value”).
dc(X) Returns the count of distinct values of the field X.
earliest(X) Returns the chronologically earliest seen value of X.
latest(X) Returns the chronologically latest seen value of X.
max(X) Returns the maximum value of the field X. If the values of X are non-numeric, the max is found from alphabetical ordering.
median(X) Returns the middle-most value of the field X.
min(X) Returns the minimum value of the field X. If the values of X are non-numeric, the min is found from alphabetical ordering.
mode(X) Returns the most frequent value of the field X.
perc(Y) Returns the X-th percentile value of the field Y. For example, perc5(total) returns the 5th percentile value of a field “total”.
range(X) Returns the difference between the max and min values of the field X.
stdev(X) Returns the sample standard deviation of the field X.
stdevp(X) Returns the population standard deviation of the field X.
sum(X) Returns the sum of the values of the field X.
sumsq(X) Returns the sum of the squares of the values of the field X.
values(X) Returns the list of all distinct values of the field X as a multi-value entry. The order of the values is alphabetical.
var(X) Returns the sample variance of the field X.

search Searches indexes for matching events. This command is implicit at the start of every search pipeline that does not begin with another generating command.
sendemail Emails search results, either inline or as an attachment, to one or more specified email addresses.

Subsearch

These are commands that you can use with subsearches.

Command Description
append Appends subsearch results to current results.
appendcols Appends the fields of the subsearch results to current results, first results to first result, second to second, and so on.
appendpipe Appends the result of the subpipeline applied to the current result set to results.
foreach Runs a templated streaming subsearch for each field in a wildcarded field list.
format Takes the results of a subsearch and formats them into a single result.
join Combine the results of a subsearch with the results of a main search.
return Specify the values to return from a subsearch.
set Performs set operations (union, diff, intersect) on subsearches.

Time

Use these commands to search based on time ranges or add time information to your events.

Command Description
gentimes Returns results that match a time-range.
localize Returns a list of the time ranges in which the search results were found.
reltime Converts the difference between ‘now’ and ‘_time’ to a human-readable value and adds adds this value to the field, ‘reltime’, in your search results.
